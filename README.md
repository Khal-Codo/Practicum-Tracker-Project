## Overview

This project defines and documents a practicum tracking web application designed through an agile business analysis approach. The solution was developed in collaboration with student developers and is structured so it can be implemented by an academic institution, subject to technical validation and approval.

The focus of the project is requirements clarity, stakeholder alignment, and iterative delivery planning.

## Objectives

* Define solution scope and stakeholder needs
* Establish a working project charter
* Produce a structured business analysis plan
* Develop agile-ready requirements documentation
* Support traceability and incremental delivery

## Artifacts Included

* Project Charter
* Business Analysis Plan
* Agile Requirements Document

## Methods Used

* Agile business analysis practices
* Stakeholder requirement modeling
* Scope and constraint definition
* Iterative requirement decomposition

## Value Demonstrated

This project demonstrates structured requirements development, agile documentation discipline, and cross-team collaboration readiness.
